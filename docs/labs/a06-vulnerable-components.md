# A06 Vulnerable and Outdated Components

## 학습 목표
- 의존성 보안 점검과 업데이트 정책을 적용한다.

## 위험 설명
- 취약한 라이브러리 사용은 애플리케이션 전체 위험으로 전파된다.

## 점검 포인트
- `build.gradle` / `package.json` 버전 고정
- CI에서 취약점 점검 가능 여부

## secure 대응
- 최소 지원 버전 유지, 정기 업데이트
- 릴리즈 노트 기반 위험 평가

## 수정 후 검증 방법
- 최신 패치 버전 반영 확인
- CI 빌드/테스트 통과 여부 확인
