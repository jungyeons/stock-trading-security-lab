# A01 Broken Access Control

## 학습 목표
- 일반 사용자와 관리자 권한 경계를 코드/설정 레벨에서 확인한다.

## 위험 설명
- 권한 검증이 누락되면 일반 사용자가 관리자 기능(종목 변경, 사용자 조회)에 접근할 수 있다.

## 점검 포인트
- 백엔드 관리자 컨트롤러의 권한 어노테이션
- 프론트 관리자 라우트 보호 로직

## secure 대응
- Spring Security role 기반 접근 제어
- 서버 권한 검증 실패 시 `403` + 감사 로그 기록

## 수정 후 검증 방법
- 일반 사용자 토큰으로 관리자 API 호출 시 `403`
- 감사 로그에 권한 거부 이벤트가 남는지 확인
