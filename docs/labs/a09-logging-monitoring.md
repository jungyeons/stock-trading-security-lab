# A09 Security Logging and Monitoring Failures

## 학습 목표
- 보안 이벤트를 추적 가능한 감사 로그로 남긴다.

## 위험 설명
- 로그인 실패/권한 거부/자산 변경 로그가 없으면 침해 탐지와 사후 분석이 어렵다.

## 점검 포인트
- 감사 로그 엔티티/저장소
- 보안 이벤트 발생 지점

## secure 대응
- 로그인 성공/실패, 권한 거부, 주문 생성, 관리자 종목 변경 기록
- 최소 필요 정보만 저장(민감정보 제외)

## 수정 후 검증 방법
- 주요 이벤트 후 감사 로그 데이터 확인
- 로그 포맷 일관성 확인
