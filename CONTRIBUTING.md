# CONTRIBUTING

## 윤리 및 사용 제한
- 본 저장소는 교육/실습 목적의 로컬 환경에서만 사용합니다.
- 외부 시스템, 제3자 서비스, 실서비스에 대한 공격/점검에 사용하지 않습니다.
- 무단 침투, 서비스 방해, 정보 탈취를 유도하는 활동을 금지합니다.

## 기여 원칙
- `secure` 브랜치 기준으로 먼저 안전한 구현과 테스트를 제안합니다.
- `vulnerable` 브랜치에는 학습에 필요한 최소 취약점만 의도적으로 반영합니다.
- 모든 보안 관련 변경은 문서(`docs/labs`, `docs/diffs`)를 함께 갱신합니다.

## 커밋 규칙
- 작은 단위로 나눠 커밋합니다.
- 예시:
  - `chore: init docs and project scaffold`
  - `feat: implement auth and profile`
  - `feat: add trading and portfolio flow`
  - `feat: add admin and audit logs`
  - `test: add backend security flow tests`
